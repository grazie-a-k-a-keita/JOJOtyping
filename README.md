## コミットメッセージルール
- feat: 新しい機能<br>
- fix: バグの修正<br>
- docs: ドキュメントのみの変更<br>
- style: 空白、フォーマット、セミコロン追加など<br>
- refactor: 仕様に影響がないコード改善(リファクタ)<br>
- perf: パフォーマンス向上関連<br>
- test: テスト関連<br>
- chore: ビルド、補助ツール、ライブラリ関連<br>
